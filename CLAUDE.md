## 1. Project Overview

### What the App Does
**BookTutor** is a client-side, Socratic active learning web application designed to help users speed-read books and achieve high information retention. The app leverages cognitive science principles, including **Active Recall**, **Spaced Repetition**, and the **Feynman Technique**.

### Core User Flow
1. **Authentication:** The user logs in via Firebase Google Authentication. Alternatively, they can toggle **Demo Mode** in the settings to preview features using a pre-configured library.
2. **Library Management:** Users view their books displayed on a 3D digital bookshelf. They can add books by:
   - Letting the AI construct a curriculum from its internal knowledge (for mainstream/bestseller books).
   - Uploading a PDF or TXT file, which is processed entirely client-side.
3. **Curriculum & Study Note Generation:** Each book is split into chapters. For each chapter, Gemini generates:
   - A **10s Takeaway** (core thesis statement).
   - A **3m Summary** (key bullet points with bolded keywords).
   - A **15m Deep Dive** (comprehensive markdown analysis).
   - A list of core **Concepts** mapped as visual nodes.
   - Probing **Flashcards** for retention reviews.
4. **Socratic Tutor Arena (Split-Pane UI):**
   - **Left Pane:** Displays summaries, deep dives, and an interactive **Concept Map**. Concept nodes turn green as they are mastered.
   - **Right Pane:** An interactive Socratic Chat that features two tabs:
     - **Teach Me:** The tutor teaches concepts step-by-step using the 80/20 rule, asks clarifying questions, quotes directly from PDF chapter texts, and outputs mastery tags (e.g., `[MASTERED: Concept Name]`) to update the concept map.
     - **Quiz & Review:** Probes student comprehension across all concepts. The tutor guides incorrect answers Socratic-style (using hints rather than revealing answers).
   - **Immersive Mode & Speech Engine:** Users can expand the chat window to full screen. A client-side Narration Engine reads the tutor's responses aloud using the browser's Web Speech API in a UK English voice.
   - **Visual Director:** In parallel with teaching, a visual sub-agent generates illustrated cartoon concepts via Pollinations.ai and renders relational Mermaid.js mindmaps.
5. **Feynman Technique Sandbox:** A dedicated environment where users select a concept and explain it as if teaching a 10-year-old. The AI grades the explanation (0-100%), details correct elements, points out gaps, and provides an ideal ELI10 model. Scores $\ge 80\%$ allow users to mark the concept as mastered.
6. **Daily Flashcard Review:** A spaced repetition deck showing due flashcards. Users rate their memory recall (Forgot, Hard, Good, Easy) to schedule future repetitions.

### Tech Stack & Architecture
- **Frontend Core:** Single Page Application (SPA) built using vanilla **HTML5**, **CSS3**, and asynchronous **ES6+ JavaScript**.
- **Styling:** Custom Vanilla CSS with variables, transitions, glassmorphic panels, and responsive grids. (No TailwindCSS).
- **Libraries & CDNs (No NPM/bundler steps required to run):**
  - **Firebase Compat SDK (v10.12.0):** Loaded via CDN for Firebase Authentication and Firestore database management.
  - **PDF.js (v3.11.174):** Used client-side to read, parse, and extract text from uploaded PDF books.
  - **Mermaid.js (v10):** Used client-side to render interactive visual mindmaps generated by the AI tutor.
  - **Web Speech API:** Leveraged for client-side TTS narration.
- **Database Routing:**
  - **Firestore (Cloud Sync):** Stores book summaries, chapter text slices, progress tracking, and chat histories under the user's specific UID.
  - **IndexedDB (Local-Only Settings):** Stores sensitive settings (e.g., the user's Google Gemini API Key and Demo Mode toggle) locally in the browser under the database `BookTutorDB` so they never leave the device.
- **AI Integrations:**
  - **LLM API:** Google Gemini API using the `gemini-2.5-flash` model. Prompts requiring structured schema leverage `responseMimeType: 'application/json'`.
  - **Image Generator:** Pollinations.ai API (free tier, prompt-based cartoon generations).

## 4. Key Architectural & Design Decisions

- **Client-Side Heavy Architecture:** The application logic is 100% client-driven. PDF text extraction, parsing, and structured data handling occur in the client, minimizing server infrastructure requirements.
- **On-Demand Curriculum Construction:** Processing a full 500-page book in one API call hits Gemini's output limit and causes timeout failures. The architecture registers a book's chapter names quickly, storing raw text slices on Firestore, and defers rich curriculum generation to the exact moment a user selects a chapter to study.
- **Context-Separated Chat History:** The `teach` and `quiz` history filters prevent context bleeding. Prompts are kept brief, instructing Gemini to return specific structures (like Socratic hints in quiz mode, or mastery triggers in teach mode).
- **JSON Mime-Types & Extraction Safeguards:** In `ai-agents.js`, Gemini calls utilize `responseMimeType: 'application/json'`. Since API JSON output is occasionally wrapped in markdown blocks or single quotes, the `queryGemini` utility employs a robust multi-stage parsing routine (stripping markdown fences, converting single quotes to double quotes, and stripping non-printable characters) to guarantee parse safety.

## 5. Data & Content Handling

### Chapter Segmentation Strategy
The function `splitPdfIntoChapters(rawText)` in [app.js](file:///c:/Users/Abzzy/.gemini/antigravity/scratch/book-tutor/app.js#L281-L329) parses the full extracted book text line-by-line using a regex evaluator to identify structural sections.
- It detects titles starting with "Chapter", "Law", "Part", "Section", "Lesson", "Habit", "Pillar", etc., followed by numbers or word forms (e.g., "One").
- It filters out noise (short page numbers/headers) and caps text chunks at 200,000 characters to remain within Firestore document limits.

### Firestore Document Paths
- **Books Metadata:** `users/{uid}/books/{bookId}`
- **Chapter Texts & Curriculum:** `users/{uid}/bookChapters/{bookId}_ch_{chapterNum}`
- **Message Histories:** `users/{uid}/chat/{chatId}`

## 7. Known Issues & Gotchas

- **Gemini Copyright & Recitation Safety:** Sending large swaths of text to Gemini can trigger safety filters (blocking content as copyrighted text recitation). Keeping chapter chunks concise and using the structure-only summary option mitigates this issue.
- **Mermaid Diagram Syntactical Strictness:** Mermaid.js will fail to render if concept labels contain syntax-breaking special characters (parentheses, brackets, hyphens). The Visual Director agent prompt instructs the AI to use simple 2-4 word alphanumeric nodes to prevent crashes, but validation sanitization should be added in the JS layer.
- **Unused Schema in IndexedDB:** During setup, the app creates `books` and `chatHistory` object stores in IndexedDB, but all user records are synced to Firestore instead. Only the `settings` store is actually utilized local-only.
